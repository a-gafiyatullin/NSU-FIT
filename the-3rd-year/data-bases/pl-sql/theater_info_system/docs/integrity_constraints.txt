Ограничения целостности на уровне БД:
    + в таблице Actor-Rank:
        + дата получения награды должна быть позже дня рождения;
        + вставка только активных работников типа "актер".
    + в таблице Employee:
        + день рождения не может быть позже даты найма.
    + в таблице Direction: 
        + вставка только активных работников типа "актер";
        + не может быть один актер на 2 роли в одном спектакле;
        + не может быть больше 1 назначения на обычную роль и 2-х назначений на главную роль.
    + в таблице Tour:
        + вставка только активных работников типа "актер", "pежиссеp-постановщик", "художник-постановщик", "диpижеp-постановщик", "музыкант";
        + дата начала не может быть позже даты конца.
    + в таблице Show:
        + ограничения на вставку различных типов активных постановщиков;
        + век спектакля не может быть раньше века жизни автора;
        + дата премьеры не может быть раньше текущего дня.
    + в таблице Repertoire:
        + запрет на добавление спектакля в репертуар, если на все роли не назначены актеры, а на главные еще и дублеры;
        + дата представления не может быть раньше премьеры спектакля.
    + в таблице Sale:
        + дата продажи автоматически заполняется.
    + в таблице Ticket:
        + не может быть два билета на одно и то же представление на одно и то же место.
    + в таблице Ticket-Subscription:
        + спектакли должны соответствовать тематике абонемента;
        + один и тот же билет не может находиться в двух разных абонементах.
    + в таблице Subscription:
        + одно из полей id_genre или id_author должно быть NULL.
    + в таблице Musician-Show:
	    + вставка только активных работников типа "музыкант".
    + каскадное удаление данных.
